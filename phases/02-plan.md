## Фазы разработки

### Фаза A — MVP: GSI + Magisk модуль (1-3 дня)
**Цель:** получить работающий прототип на телефоне

1. Узнать модель телефона и проверить Treble-совместимость
2. Скачать готовый LineageOS 22.2 GSI (ARM64 + AB)
3. Создать Magisk-модуль `ruslanos-setup`:
   - Устанавливает Ruslan Agent в `/system/priv-app/RuslanAgent/RuslanAgent.apk`
   - Создаёт сервис автозапуска (init.d или Magisk service)
   - Блокирует стоковые приложения через `.replace` файлы
   - Настраивает системные проперти
4. Создать лаунчер RuslanShell (APK):
   - Простой экран с логотипом и кнопкой «Чат с Русланом»
   - Lock Task Mode (киоск)
   - Открывает Telegram WebView или локальный веб-интерфейс
5. Зашить GSI, установить Magisk, поставить модуль

### Фаза B — Полная сборка LineageOS + RuslanOS (2-4 недели)
**Цель:** собственный ROM с Русланом в системе

1. Развернуть билд-среду (Mac Mini M4 через OrbStack, или HES)
2. Форкнуть LineageOS 22.2 исходники
3. Создать device tree для телефона (если нет официального)
4. Добавить Ruslan Agent в `packages/apps/RuslanAgent/`
5. Собрать первый образ
6. Кастомизировать:
   - Свой bootanimation
   - Заменить лаунчер по умолчанию
   - Отключить ненужные системные приложения
7. Настроить OTA-обновления

### Фаза C — Экосистема (долгосрочно)
1. Свой репозиторий пакетов
2. Механизм «модификация через Руслана» (команды → system API)
3. Голосовое управление ОС
4. OTA-инфраструктура

## Технические решения

### Почему GSI первый?
- Не требует сборки — готовый образ от LineageOS
- Работает на любом Treble-устройстве
- Позволяет прототипировать модуль и лаунчер без билд-фермы
- Переход к полной сборке — только когда прототип готов

### Языки и инструменты
- **RuslanShell (лаунчер):** Kotlin + Jetpack Compose (Android Studio)
- **Magisk модуль:** shell-скрипты + update-binary
- **CI/CD:** GitHub Actions (компиляция APK, подготовка модуля)
- **Сборка ROM:** repo, make, soong, ccache

### Лицензия
- Apache 2.0 (как LineageOS)
- Весь код RuslanOS — Apache 2.0
- Attribution сохранён

## Риски
1. **16GB RAM на Mac Mini** — сборка может упираться в память → решение: -j2, ccache, swap
2. **Нет официального device tree** — портирование может быть нетривиальным
3. **Lock Task Mode** — требует Device Policy Controller (решение: через ADB `dpm set-device-owner`)
4. **Telegram API ключи** — нужно управлять безопасно
